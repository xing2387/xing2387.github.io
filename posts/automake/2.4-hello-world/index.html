2.4.2 configure.ac解释#

configure.ac的写法见Writing configure.ac

configure.ac的内容：

AC_INIT([amhello], [1.0], [bug-automake@gnu.org])
AM_INIT_AUTOMAKE([-Wall -Werror foreign])
AC_PROG_CC
AC_CONFIG_HEADERS([config.h])
AC_CONFIG_FILES([
 Makefile
 src/Makefile
])
AC_OUTPUT

AC_开头的宏是Autoconf的宏Autoconf Macro Index，AM_开头的是Automake的宏Macro Index (automake)。

前两行初始化Autoconf和Automake。

  • AC_INIT ：定义目标名称、版本、联系方式。联系方式在./configure –help中输出。

  • AM_INIT_AUTOMAKE：设定Automake的选项，见Options (automake)，-Wall和-Werror是设置警告的输出，foreign是避免因为缺少GNU标准需要的ChangeLog等文件而complain。

  • AC_PROG_CC：让configure脚本去搜索并设置CC。

  • AC_CONFIG_HEADERS： 指定configure脚本生成一个头文件config.h

  • AC_CONFIG_FILES：指定configure脚本需要根据*.in模板生成对应的文件，Makefile都要加到这个列表中。

  • AC_OUTPUT：相当于“开炮”。

2.4.3 Makefile.am 解释#

文件src/Makefile.am内容：

bin_PROGRAMS = hello
hello_SOURCES = main.c

文件Makefile.am内容：

SUBDIRS = src
dist_doc_DATA = README

  • automake几乎只是将Makefile.am复制到Makefile.in中，除了添加一些编译需要的变量。

  • 语法同常规的Makefile相同。

  • 一般只是定义一些变量。

  • _PROGRAMS 后缀指定应该编译的目标，并且应该安装到${bindir}中. 类似的还有_SCRIPTS, _DATA, _LIBRARIES等。可以在它们前面指定安装目录，如dist_doc_DATA

  • SUBDIRS 指定子目录，让make先递归进去处理，并且make install时也先安装里面的目录的产物。

  • dist_doc_DATA = README：在make dist时，_Data的东西不会被自动打进tar包中，而dist_前缀用于明确要打进去，_Data前的doc_指定安装到${docdir}中。
