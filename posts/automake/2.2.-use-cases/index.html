[gnu-automake-manual] 2.2. Use Cases

July 2, 2021 · 2 min · Me | Suggest Changes

为了适应不同系统环境下的编译，需要有.configure命令：run ./configure && make && make install

2.2 Use Cases#

2.2.2 make 的目标(targets)#

命令
make checkRun the test suite, if any.
make installcheckCheck the installed programs or libraries, if supported.
make distcleanAdditionally erase anything ./configure created.
make distRecreate package-version.tar.gz from all the source files.

2.2.3 目录变量#

部分变量：

Directory variableDefault value
prefix/usr/local
exec_prefix${prefix}
bindir${exec_prefix}/bin
libdir${exec_prefix}/lib
includedir${prefix}/include
datarootdir${prefix}/share
datadir${datarootdir}
mandir${datarootdir}/man
infodir${datarootdir}/info
docdir${datarootdir}/doc/${PACKAGE}

使用： ./configure --prefix ~/usr

2.2.4 Configuration变量#

参数
CCC compiler command
CFLAGSC compiler flags
CXXC++ compiler command
CXXFLAGSC++ compiler flags
LDFLAGSlinker flags
CPPFLAGSC/C++ preprocessor flags

使用：./configure --prefix ~/usr CC=gcc-3 CPPFLAGS=-I$HOME/usr/include LDFLAGS=-L$HOME/usr/lib

2.2.5 默认Configuration配置#

新建 prefix/share/config.site ： 如 prefix == ~/usr 则新建 ~/usr/share/config.site，填入一下内容：

test -z "$CC" && CC=gcc-3
test -z "$CPPFLAGS" && CPPFLAGS=-I$HOME/usr/include
test -z "$LDFLAGS" && LDFLAGS=-L$HOME/usr/lib

则 ./configure --prefix ~/usr CC=gcc-3 CPPFLAGS=-I$HOME/usr/include LDFLAGS=-L$HOME/usr/lib 可通过默认参数省略成 ./configure

2.2.6 独立build目录#

build目录是执行./configure时的目录。 新建新目录来执行configure就可以将编译产物和源码分开，也可以用于以不同设置来编译源码。

2.2.7分开安装数据和可执行文件#

当两台机器通过网络文件服务sshfs等共享源代码，但单独在各自本地目录编译安装时，可以选择将数据安装在公共目录，只单独安装可执行程序在各自本地目录，以适应各自不同的系统环境。 make install 可以看作是 make install-exec 并 make install-data的快捷方式。

若 /usr/share是两台机器共享的目录，/usr是各自的本地目录。 则 ：

  • A机器可以 正常的make install，将可执行程序放到本地/usr中，将数据放到共享的/usr/share中。 （若指定了–prefix /usr，且默认exec_prefix为${prefix}，datarootdir为${prefix}/share）

  • B机器只需执行 make install-exec，将可执行程序放到本地/usr中。

2.2.8 交叉编译#

在./configure时有参数指定编译和运行的平台：

参数
–build=build执行编译的平台
–host=host运行编译产物的平台

如 ./configure --build i686-pc-linux-gnu --host i586-mingw32msvc ，是在i686-pc-linux-gnu 平台编译可在 i586-mingw32msvc平台运行的程序。

在编译交叉编译工具时，需要用–target=target指定编译的编译工具的目标平台。

2.2.9 安装时重命名#

在安装时如果跟已有程序文件名冲突或单纯只是想重命名的话，可以指定可执行文件

参数
–program-prefix=prefix可执行文件添加前缀
–program-suffix=suffix可执行文件添加后缀
–program-transform-name=program用sed命令处理文件名

例： ./configure --program-prefix test- 会将hello安装到/usr/local/bin/test-hello 例： --program-transform-name='s/^/g/; s/^gg/g/; s/^gless/less/'

2.2.10 重定向安装目录-DESTDIR#

当你想知道安装了什么东西或者想手动安装时，可以用 DESTDIR参数指定安装目录。 如 make DESTDIR=$HOME/inst install ， 原本安装在 /usr/bin的hello，会安装到 ~/inst/usr/bin/hello， 随后就可以手动复制到其他地方安装。

2.2.12 自动跟踪依赖#

configure参数： –disable-dependency-tracking 禁止 –enable-dependency-tracking
